⚙️ Cómo configurarlo paso a paso

Como el archivo se abre localmente (file://), Google no permite autenticarse así directamente. La solución más sencilla es publicarlo gratis en GitHub Pages — tarda menos de 5 minutos y tus alumnos acceden con un enlace normal.

1
Crea una cuenta en GitHub
Ve a github.com y regístrate gratis (si ya tienes cuenta, sáltate este paso).
2
Sube el archivo HTML
Crea un repositorio nuevo → ponle el nombre que quieras → haz clic en "uploading an existing file" → arrastra este archivo HTML → nómbralo exactamente index.html → pulsa "Commit changes".
3
Activa GitHub Pages
En tu repositorio ve a Settings → Pages → Source, selecciona la rama main y pulsa Save. En unos segundos tendrás una URL del tipo https://tunombre.github.io/nombre-repo/. ¡Esa es la URL que envías a tus alumnos!
4
Configura Google Cloud
Ve a console.cloud.google.com, crea un proyecto y activa la Google Drive API en "APIs y servicios → Biblioteca".
5
Crea las credenciales OAuth
En "APIs y servicios → Credenciales" crea un ID de cliente OAuth 2.0 de tipo "Aplicación web". En Orígenes de JavaScript autorizados añade tu URL de GitHub Pages (ej: https://tunombre.github.io). Copia el Client ID que te genera.
6
Pega el Client ID en el código
Abre el archivo HTML con un editor de texto (Notepad, TextEdit…), busca TU_CLIENT_ID_AQUI y sustitúyelo por tu Client ID. Repite el paso 2 para subir la versión actualizada.
7
Añade el Folder ID de Drive
En Google Drive crea una carpeta para las entregas. Ábrela y copia el ID que aparece al final de la URL: drive.google.com/drive/folders/ESTE_ES_EL_ID. Pégalo en el código donde pone TU_FOLDER_ID_AQUI y vuelve a subir el archivo.
💡 Tip: cuando todo esté configurado, el aviso naranja desaparece automáticamente. Las entregas se organizarán en subcarpetas con el formato Semana X — Curso — Tipo — Nombre alumno.
